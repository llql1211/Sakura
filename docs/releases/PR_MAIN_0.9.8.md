# PR：发布 Sakura 0.9.8

## 建议标题

`feat: 发布 Sakura 0.9.8`

## 合并方向

`dev` → `main`

## 这次合并解决什么

`0.9.8` 不是单点功能更新。它把 `0.9.7` 之后分散在对话等待、上下文、长期记忆、TTS、插件、设置页和存储层的改动收成一个可发布版本。

用户最容易感知的变化有四项：等待模型回复时会先出现角色化短回应；输入栏可以框选截图；新会话能接住上一段对话；角色包可以通过图形化工作室编辑和导出。底层则补齐资源回收、配置迁移、原子写入与启动自检，重点处理退出崩溃、设置保存失败和覆盖升级的数据风险。

## 用户侧变化

- 新增本地快速接话，支持规则模式、模型增强、触发延迟、概率和语音预生成。正式回复开始后会取消未完成的接话任务，避免接话堵住 TTS 队列。
- 输入栏新增区域截图，可附加到下一条消息；高 DPI、多显示器和 Retina 下按实际像素处理，修复模糊与选区错位。
- 点击桌宠可呼出输入栏。设置、历史和日志窗口有独立任务栏入口及最小化按钮，并会跟随桌宠的置顶状态。
- 舞台宽度改为跟随内容，默认按立绘和气泡的实际区域生成碰撞遮罩，透明空白不再大面积挡住桌面点击。
- 设置与历史窗口减少阻塞式加载；记忆列表支持筛选、批量删除、新增和编辑，不再限制显示条数。
- 新增 `start_studio.bat` 和 SakuraCharacterStudio，可编辑角色基础信息、人格卡、主题、立绘、语气及语音资源并导出 `.char`。

## Agent、上下文与记忆

- 引入 `ContextOrchestrator`，把时间、近期对话、屏幕摘要、长期记忆和插件上下文统一转换为带优先级、信任级别、生命周期及 token 预算的片段。
- 提示词拆成静态角色约束与每步动态上下文，减少工具循环中重复发送不变内容。
- 新增最近会话上下文。在当前会话刚开始且消息较少时，从持久化聊天记录提取上一段有效对话；达到当前会话消息阈值后停止注入。
- 长期记忆增加核心、语义、情景、流程和会话层级；自动整理可新增、更新、去重和处理冲突，召回服务按本轮输入与视觉摘要选择少量相关记忆。
- 工具循环的每轮步数、每步调用数和整轮调用总数可配置，并在读取时限制到安全范围。

## 架构与可靠性

- 统一 `StoragePaths`，配置、聊天记录、记忆、日志、缓存和迁移备份不再各自拼路径。YAML 与关键状态写入改用原子替换，并保留上一份 `.bak`。
- 加入配置版本迁移和应用版本记录。迁移成功后才推进版本号，失败会保留备份并在下次启动重试。
- 加入单实例锁和启动自检，提前报告数据目录不可写、磁盘空间不足、配置不可访问及记忆库残留锁。
- 引入 `ResourceManager` 管理 Qt Worker、Python 线程、进程、MCP、记忆、插件和 TTS 的关闭顺序，避免 Qt 已销毁后后台任务继续回调。
- 将 TTS 拆为配置、工厂、服务监管、合成队列、播放端点和共享类型；修复 broken pipe 自动恢复、无效文本合成、设置未变时重复重建 Provider、退出时原生崩溃等问题。
- 发布工作流明确排除 `data/`，避免覆盖升级替换用户数据；新增默认配置生成和 `tools/cleanup.py` 安全清理工具。

## 插件与公开接口

- 插件入口统一到 `app.plugins.*`，支持工具、设置页、事件监听、上下文提供者和渲染器贡献点。
- 插件管理页显示版本、权限、状态和错误，并将插件自己的设置放到独立窗口。
- 移除旧的仓库级 `sdk/`、Shinsekai 兼容层和示例插件。现有第三方插件若仍导入 `sdk.*`，需按 `docs/SAKURA_PLUGIN_SDK.md` 迁移。
- 接话资源采用版本化 intent 契约；角色清单缺失或不兼容时直接停用接话，不影响普通对话。

## 兼容性与升级

- 版本号从 `0.9.7-dev` 更新为 `0.9.8`。
- Windows 仍使用发布包内的 `runtime/python.exe`；启动和安装脚本不会回退到系统 Python。
- TTS 文本语言固定为日语，旧语言控件已移除。
- `mcp.yaml`、`plugins.yaml` 不再随仓库发布，首次启动会生成默认配置，旧数据通过迁移框架保留。
- 推荐覆盖解压完整包并重新运行 `install.bat`。发布包没有 `data/`，但升级前仍建议备份 `data/`、自制角色和自定义插件。

## 风险点

- 本次合并涉及 UI 主窗口、TTS 生命周期、配置迁移、记忆结构和插件公开接口，回归重点应放在启动/退出、保存设置、切换角色、连续对话、工具确认、TTS 开关和旧数据升级。
- 模型增强接话需要本地 `BAAI/bge-small-zh-v1.5`；未安装时可使用规则模式，不应阻断普通回复。
- 第三方旧插件可能因 `sdk.*` 移除而无法加载，插件管理页应显示失败原因。

## 当前验证结果

- `runtime/python.exe -m pytest --basetemp .pytest-tmp-0.9.8`：`1187 passed, 3 skipped, 5 errors`。
- 5 个 error 都发生在 `tests/ui/test_history_window.py` 的 setup 阶段：当前发布 runtime 未安装 `pytest-qt`，因此没有 `qtbot` fixture。没有测试断言失败。
- pytest 还提示 `env`、`qt_api`、`timeout` 和 `timeout_method` 配置项缺少对应插件。发布前 CI 仍需跑完完整依赖环境中的测试。
