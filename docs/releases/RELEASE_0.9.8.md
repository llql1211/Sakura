# Sakura 0.9.8

## 本地快速接话

这是这个版本最容易被感知到的变化。

之前发完消息，Sakura 在等模型生成完整回复的那几秒到十几秒里完全沉默，什么反应都没有。现在她可以先接一句——会根据当前角色语气和聊天上下文，从角色声线匹配的短回应清单里挑一句，用角色语音说出来。

接话有两种模式：

- **规则模式**：基于角色清单的 intent 匹配，不需要额外模型，开箱就能用。
- **模型增强**：调用本地向量模型（`BAAI/bge-small-zh-v1.5`）做语义筛选，匹配更准。没装的话会自动退到规则模式，不影响正常聊天。

可以在设置里控制延迟（等多长时间后接话）、触发概率和是否启用语音播报。正式回复一开始生成，未完成的接话会自动取消，不会堵住 TTS 队列。不喜欢这个功能直接关掉就行。

角色作者也可以给 `.char` 包带自定义接话清单和语音文件，格式见 `assets/backchannels/MANIFEST_GUIDE.md`。

## 长期记忆

记忆系统这一版改得比较多。

之前记忆不分类型，全塞在一起，找也费劲。现在改成分层组织：

| 层级 | 存什么 | 例子 |
|---|---|---|
| 核心信息 | 名字、喜好、身份 | "主人叫小明，喜欢喝咖啡" |
| 经历 | 发生过的事 | "上周一起看了《天气之子》" |
| 任务 | 当前在做的事 | "正在帮主人整理论文参考文献" |
| 操作习惯 | 经常做的事 | "主人每天下午三点会喝一杯拿铁" |
| 会话 | 本轮短暂记录 | 当前对话的临时信息 |

其他改进：

- **召回**：Sakura 会根据本轮输入和视觉摘要，自动找相关记忆放进上下文。跟当前话题无关的记忆不会占 token。
- **自动更新**：对话过程中 Sakura 可以新增和更新记忆。整理逻辑会处理重复内容，冲突信息会尝试合并或保留较新的。
- **记忆管理页**：现在可以拉高列表、按层级筛选、批量选择和删除，也可以直接新增和编辑条目的文本与层级。之前显示的条数有上限，现在已经去掉了。

## 插件系统

插件从接口到管理整个重写了一遍。

**对用户来说**，最直接的变化是插件管理页。现在能看到每个插件的名字、版本、权限和运行状态（正常 / 已禁用 / 有错误）。插件自己的设置也独立开了个对话框，不再跟系统设置挤在一起。

**对开发者来说**，插件入口统一到了 `app.plugins.*`。插件现在可以注册这些东西：

- 工具（`builtin_tools`）—— Agent 可以调用的新能力
- 设置页（`setting_pages`）—— 插件的 UI 配置界面
- 事件监听（`runtime_event`）—— 在对话运行的不同阶段插入逻辑
- 上下文提供者（`context_provider`）—— 给 Agent 的上下文注入自定义信息
- 渲染器（`renderer`）—— 自定义立绘和舞台渲染方式

旧的 `sdk.*` 和 Shinsekai 兼容层已经移除。如果你的第三方插件还在用这些接口，升级后插件管理页会显示加载失败和原因。插件作者需要看 [Sakura 插件 SDK 文档](https://github.com/Rvosy/sakura/blob/main/docs/SAKURA_PLUGIN_SDK.md) 来迁移。

Playwright 浏览器插件继续随项目提供，可以在设置里开启。

## 角色工坊（SakuraCharacterStudio）

Windows 用户双击 `start_studio.bat` 就能打开。这是一个图形化的角色编辑器，之前做角色包要手写 JSON 和手动拼文件，现在用界面就能搞定。

能做的事：

- 新建角色，填写基础信息（名称、简介等）
- 编辑人格卡（性格、人设、说话风格等文本）
- 搭配主题色，实时预览
- 导入立绘（支持多套立绘切换）
- 关联语气标签和语音参考音频
- 导入 GPT-SoVITS 权重文件
- 校验并导出 `.char` 文件，发给别人也能直接用 Sakura 导入

工作室用的是 Sakura 自带的 `runtime`，不用另外装 Python。保存的 `.char` 文件与 Sakura 主程序完全兼容。

## 上下文和会话续接

之前每次启动 Sakura 都是"全新"的。关掉再开，她完全不记得刚刚聊过什么。

现在有了会话上下文续接：新会话开始时，Sakura 会从存储的聊天记录里拉出上一段有效对话，生成一个简短摘要放进当前上下文。在当前会话消息多起来之后，这段续接内容会自动退出，不再占 token。

另外上下文编排也改过了。`ContextOrchestrator` 统一管理所有上下文的优先级、信任级别和 token 预算，不重要的内容不会挤掉核心信息。提示词也拆分成了静态角色约束和动态上下文，工具循环里不再一遍遍重复不变的内容。

工具循环的步数和调用上限现在可以在设置里调整，默认值够日常用，想放开也有安全范围兜底。

## 截图修复

高 DPI、多显示器和 macOS Retina 下的截图错位和模糊问题已经修复。截出来的图现在按实际像素处理，选区和光标位置都跟屏幕对得上。

> 注意：输入栏的框选截图按钮是已有的功能，这个版本没有新增按钮。改动只是让截出来的图在不同缩放下不再错位和模糊。

## TTS 稳定性

TTS 这次从架构上拆了一层。原来是近三千行的单文件，现在拆成了：

- **配置和 Provider 工厂**：管理本地、外置和空 Provider 的创建
- **服务监管**：启动、健康检查、自动重连
- **合成队列**：排队、限流、取消和错误处理
- **播放端点**：音频输出和队列管理

修的主要问题：

- 本地 GPT-SoVITS 服务断开后的 broken pipe 不会再让程序卡死
- 切换 TTS 设置时不会再重复重建 Provider
- 退出程序时的原生崩溃概率大幅降低
- 不可发音的文本不会再被送进 TTS 导致报错
- 无效回复不会再误触发语音播报

## 窗口和交互

- 点击桌宠可以直接呼出输入栏。回复结束后，输入栏按设置正常收起。
- 设置、聊天历史和运行日志窗口现在有独立的任务栏入口和最小化按钮。
- 桌宠置顶时，子窗口会跟随置顶，不会被桌宠自己挡住。
- 立绘周围的透明区域默认可以点击穿透，桌面上随便点不会误触到桌宠。

## 升级方法

1. 关闭 Sakura。
2. 备份原目录的 `data/`（虽然发布包不会覆盖，备份是个好习惯）。自制角色和插件也建议一并备份。
3. 从 [Releases](https://github.com/Rvosy/sakura/releases) 下载 `sakura-v0.9.8-windows-x64.zip`。
4. 解压后把内容复制到原 Sakura 目录，遇到同名文件选择覆盖。
5. 重新运行 `install.bat`。
6. 启动 Sakura。首次启动会自动执行配置迁移，迁移前的文件会备份到 `data/migration_backup/`。

`0.9.8` 发布包不包含 `data/`，所以覆盖的时候你的 API 配置、聊天记录、长期记忆、角色数据和 TTS 文件都不会被替换。也不要手动删除原目录里的 `data/`。

如果升级后想清理旧版缓存，可以先跑到 `runtime/python.exe tools/cleanup.py` 预览，确认没问题再加 `--apply` 执行。

## 兼容性注意

- 旧版第三方插件如果使用 `sdk.*` 或兼容层接口，升级后无法加载。插件作者需要改用 `app.plugins.*`，迁移写法见[插件 SDK 文档](https://github.com/Rvosy/sakura/blob/main/docs/SAKURA_PLUGIN_SDK.md)。
- TTS 文本语言已固定为日语，设置页不再提供语言选项。
- `mcp.yaml` 和 `plugins.yaml` 不再随仓库发布，首次启动自动生成。已有配置不受影响。

完整变化见 [CHANGELOG.md](https://github.com/Rvosy/sakura/blob/main/CHANGELOG.md)。
